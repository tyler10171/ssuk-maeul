## Phase 1: MVP (예상 4~6주)

### 목표
"내 조건에 맞는 정부·지자체 지원금을 놓치지 않고 알림으로 받는다."
가입한 부모가 자신의 지역·시기·자녀나이·소득·주택·자격 정보를 입력하면, 받을 수 있는 지원 목록을 즉시 확인하고, 신청 시기가 다가오면 이메일·웹푸시로 자동 알림을 받는다.

### 기능
- [ ] 이메일 회원가입 / 로그인 / 비밀번호 재설정
- [ ] 사용자 프로필 (시도·시군구, 시기, 자녀, 소득분위·가구원수·월소득(선택)·주택보유·특수자격)
- [ ] 자녀 1명 등록 (생년월일 또는 출산예정일)
- [ ] 정책 통합 탐색 페이지 (필터: 시기/지역/카테고리/지원유형, 검색어)
- [ ] 정책 상세 페이지 (대상조건, 지원내용, 신청방법, 외부 신청 링크)
- [ ] 조건 매칭 알고리즘 (서버에서 일 1회 + 프로필 변경 시 실행)
- [ ] "나의 추천 지원" 페이지 (매칭 결과 + D-Day 표시)
- [ ] 매칭 상태 변경 (추천 → 관심 / 무관 토글, 완료는 Phase 2에서 활성화)
- [ ] 이메일 자동 발송 (신규 매칭 / 마감 임박 D-7,D-3 / 시기 도래)
- [ ] 웹 푸시 알림 등록·발송 (브라우저 권한 흐름)
- [ ] 사이트 내 알림함 (읽음/안읽음)
- [ ] 알림 설정 페이지 (채널 on/off, 카테고리별 on/off)
- [ ] 정책 데이터 자동 동기화 스케줄러 (Vercel Cron + Supabase Edge Function)
- [ ] 관리자 대시보드 (정책 수동 등록·수정, 동기화 상태, 사용자 통계)
- [ ] 반응형 디자인 (모바일 우선)

### 데이터
- User, Child, Policy, Match, Notification

### 인증
- 이메일 + 비밀번호 (Supabase Auth)
- 이메일 인증 링크 발송

### 정책 데이터 범위
- 중앙정부 임신·출산·영유아 정책 30건+ (수동 등록 + 가능한 API)
- 서울특별시 25개 자치구 중 우선 5개 구 (강남·서초·송파·마포·성동)
- 17개 광역 확장은 운영 안정화 후 점진적

### "진짜 제품" 체크리스트
- [ ] 실제 Supabase Postgres 연결 (목업 데이터 X)
- [ ] 실제 Supabase Auth (하드코딩 비밀번호 X)
- [ ] Resend로 실제 이메일 발송 (콘솔 로그만 X)
- [ ] Vercel에 실제 배포 (localhost X)
- [ ] 외부 사용자가 URL로 접속해서 가입·매칭·알림 수신까지 완주 가능
- [ ] 환경변수는 .env.local + Vercel 환경변수에 분리

## Phase 2: 확장 (예상 3~4주, MVP 안정화 후)

### 전제 조건
- Phase 1이 Vercel + Supabase에 안정 배포된 상태
- 활성 사용자 100명 이상 또는 정책 등록 200건 이상

### 목표
"신청의 실제 부담을 줄인다."
정책별로 필요한 서류·제출처를 단계별로 안내하고, 사용자가 자녀를 여러 명 관리하며 수령 진행 상황을 트래킹할 수 있다.

### 기능
- [ ] 정책별 문서 작성 체크리스트 (정책 데이터 모델에 documents 필드 추가)
- [ ] 제출처 안내 + 원클릭 신청 링크
- [ ] 자녀 여러 명 관리 (UI에서 자녀 추가/전환)
- [ ] 정책 즐겨찾기 (Match.status = "관심" 활용)
- [ ] 수령 완료 체크 (Match.status = "완료" 활성화) + "이번 달 받은 지원금" 요약
- [ ] 소셜 로그인 (카카오 + 구글 OAuth)
- [ ] 모바일 반응형 강화 + 푸터/네비 개선
- [ ] 정책별 후속 알림 (수령 후 다음 단계 안내)

### 추가 데이터
- Policy 테이블에 documents (JSON 배열: 서류명, 발급처) 컬럼 추가
- Policy 테이블에 submission_places (JSON 배열) 컬럼 추가
- User에 social_provider 컬럼 추가 (소셜 로그인 식별)

### 통합 테스트
- Phase 1 기능이 여전히 정상 동작하는지 확인
- 기존 사용자의 매칭 결과가 깨지지 않는지 확인
- 알림 발송이 끊김 없이 계속되는지 확인

## Phase 3: 고도화 (예상 장기, Phase 2 안정화 후)

### 전제 조건
- Phase 1 + 2가 안정 운영 중
- 활성 사용자 1,000명 이상 또는 정책 등록 500건 이상

### 목표
"커뮤니티와 지자체 종합 플랫폼으로."

### 기능
- [ ] 카카오톡 알림톡 연동 (사업자등록 + 채널 심사 필요)
- [ ] 지자체 시설 정보 (어린이집/키즈카페/주말어린이집)
- [ ] 정책별 후기 · 커뮤니티
- [ ] AI 정책 상담 챗봇 (Claude API 등 활용)
- [ ] PWA 설치 (홈화면 아이콘, 오프라인 일부 지원)
- [ ] 관리자 통계 대시보드 (KPI: 활성 사용자/매칭률/알림 도달률)
- [ ] 정책 변경 추적 히스토리 (policy_history 테이블)
- [ ] 다국어 지원 (외국인 부모용 영어)

### 주의사항
- **알림톡**: 비용 발생 (건당 약 10원), 사업자등록·채널 심사 시간 필요.
- **AI 챗봇**: API 비용 누적. 캐싱 + 비용 상한 필수.
- **다국어**: 정책 텍스트 번역 비용/유지보수.
