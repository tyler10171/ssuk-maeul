# 쑥 마을 -- PRD (Product Requirements Document)

> 생성일: 2026-06-15
> 생성 도구: Show Me The PRD

## 1. 제품 개요

### 한 줄 요약
부모가 임신부터 육아 전 과정에서 받을 수 있는 정부·지자체 지원금을 놓치지 않도록, 본인 조건에 맞는 지원을 자동으로 찾아주고 시기마다 알려주는 웹 플랫폼.

### 해결하는 문제
- 임신·출산·육아 단계마다 정부·지자체 지원 정책이 너무 많고 자주 바뀌어서, 부모가 자신이 받을 수 있는 게 뭔지 파악하기 어렵다.
- 신청 시기를 놓치면 그대로 못 받는다. 시기를 챙겨주는 곳이 거의 없다.
- 정책 정보가 정부24·복지로·시도청·구청 등에 흩어져 있어 한 곳에서 비교가 안 된다.
- 결과적으로 "있는데 못 받는 지원"이 매년 누적된다.

### 핵심 가치
- **놓치지 않게 한다**: 검색을 안 해도 시기·조건에 맞는 정책이 알아서 알림으로 온다.
- **내 조건 기준**: 거주지·시기·자녀나이·소득분위·주택·특수자격까지 맞춰 진짜로 받을 수 있는 것만 보여준다.
- **중앙 + 17개 광역 + 시군구 통합**: 정부24처럼 중앙만 보는 게 아니라 내가 사는 동네 혜택까지 한눈에.
- **자동 최신화**: 공공데이터포털 API로 정책 변경을 자동 동기화해 정보가 늘 최신이다.

### 기존 대안과의 차이
| 기존 | 한계 | 쑥 마을 |
|------|------|---------|
| 정부24 / 복지로 | 검색 위주, 알림 약함, 시기 안내 없음 | 조건 매칭 + 시기별 자동 알림 |
| 지자체 별 사이트 | 흩어져 있고 비교 어려움 | 한 곳에 통합 |
| 임신·육아 앱 (마미톡 등) | 발달 정보 중심, 지원금 약함 | 지원금 전문 |

## 2. 사용자

### 주요 사용자
- **누구**: 대한민국에 거주하는 임신·출산·영유아 자녀(0~만 8세 추정)를 둔 부모. 1차로 임신 중~만 2세 자녀 부모에 집중.
- **상황**: 산부인과 진료 후, 출생신고 직후, 아이가 새 단계(돌·어린이집·초등 등)에 진입할 때. 보통 모바일에서 짬짬이 확인.
- **목표**: "내가 지금 받을 수 있는 지원이 뭐고, 언제까지 어떻게 신청하면 되는지" 한 번에 파악.

### 사용자 시나리오
1. 임신 20주차의 김민지 씨가 산부인과에서 옆 산모가 "교통비 지원 받았어요?"라고 물어보는 걸 듣고 쑥 마을에 가입한다.
2. 거주지(서울 강남구), 임신주차, 출산예정일, 소득분위(5분위), 주택(전세), 특수자격(해당없음)을 입력한다.
3. "나의 추천 지원" 페이지에 임산부 교통비 70만원, 첫만남이용권 200만원, 강남구 임산부 영양제 지원 등 12건이 뜬다. 각 항목에 신청 마감일이 D-7, D-23 식으로 표시된다.
4. 일주일 후 출산예정일이 다가오자 이메일과 사이트 알림함에 "출산 1개월 전, 행복출산 원스톱 서비스 신청 시기입니다" 알림이 자동 도착한다.
5. 알림 링크를 누르면 신청 안내 페이지로 이동, 외부 정부24 신청 페이지 링크로 바로 이동한다.

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름 (신규 사용자 → 첫 알림 수신까지)
```
랜딩 -> 회원가입(이메일/비번) -> 프로필 입력 -> 자녀 등록 -> 나의 추천 지원 확인 -> (며칠 후) 시기별 알림 자동 수신
```

### 상세 흐름

#### A. 신규 가입 & 프로필 작성
1. **랜딩**: 가치 소개 + "내 지원금 찾기 시작하기" CTA
2. **회원가입**: 이메일 + 비밀번호. 가입 직후 이메일 인증 링크 발송
3. **프로필 1단계 (필수)**: 시도 → 시군구, 현재 시기(임신/출산/육아) 선택
4. **프로필 2단계 (필수)**: 자녀 1명 등록 -- 임신 중이면 출산예정일, 그 외엔 생년월일
5. **프로필 3단계 (선택, 매칭 정확도 향상)**: 소득분위/가구원수/월소득(선택)/주택보유상태/특수자격(다자녀·한부모·장애·외국인 등)
6. **나의 추천 지원**: 입력 조건 기반 매칭 결과 즉시 표시

#### B. 정책 탐색
1. 상단 검색바 또는 필터(시기/지역/카테고리/지원유형)
2. 결과 리스트 -- 카드형으로 정책명·지원금액·D-Day·기관 표시
3. 카드 클릭 → 상세 페이지: 대상 조건, 지원 내용, 필요 서류, 신청 방법, 외부 링크

#### C. 알림 수신
1. 시스템이 매일 1회 cron 실행: 사용자별 임박 정책 계산
2. 해당하는 사용자에게 이메일 + 웹푸시 발송 (사용자가 켠 채널만)
3. 사이트 내 알림함에도 동일 기록 적재
4. 알림 클릭 → 정책 상세 페이지

#### D. 관리자 흐름
1. 관리자 로그인 (별도 권한)
2. 대시보드: API 동기화 마지막 시각, 신규 정책 수, 오류 로그
3. 정책 목록에서 수동 등록·수정·비활성화

## 5. 성공 기준

### Phase 1 (MVP) 성공 기준
- [ ] 가입한 사용자의 90% 이상이 프로필 완성 후 "나의 추천 지원" 페이지에 1건 이상의 매칭 결과를 받는다.
- [ ] 중앙정부 임신·출산·영유아 정책 30건 + 서울특별시 25개 자치구 중 최소 5개 구의 정책이 등록되어 있다.
- [ ] 정책 데이터가 매일 1회 자동 동기화되고, 동기화 실패 시 관리자에게 알림이 간다.
- [ ] 사용자가 이메일과 웹푸시 알림을 실제로 받고, 알림 클릭률(이메일 열람율)이 30% 이상이다.
- [ ] 신청 마감 임박 알림(D-7, D-3)이 자동 발송되어 누락 없이 도달한다.

### Phase 2 이후 지표
- 수령 완료 체크 기능을 통해 매칭된 정책 중 사용자가 실제 신청한 비율(self-reported) 측정.
- 한 사용자가 "쑥 마을 덕에 받은 지원금 총액" 표시.

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다.
> AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- **문서 작성 가이드·제출처 원클릭 신청** -- Phase 2로 분리. 정책별 콘텐츠 작성 부담이 매우 크고 정책 변경 시 유지보수 어려움.
- **카카오톡 알림톡** -- Phase 3. 사업자등록·채널 심사 필요. MVP에선 이메일+웹푸시로 검증.
- **소셜 로그인(카카오/구글)** -- Phase 2. MVP는 이메일+비밀번호 단일.
- **자녀 여러 명 관리** -- Phase 2. MVP는 1자녀.
- **커뮤니티·후기·AI 챗봇** -- Phase 3.
- **모바일 앱(네이티브)** -- 웹 반응형으로 충분. PWA 설치는 Phase 3.
- **결제 / 유료 프리미엄** -- 비즈니스 모델 검증 후 검토.
- **임신 외 생애주기(취학 아동·청소년)** -- 1차 타깃은 임신~만 2세.
- **지자체 시설 정보(어린이집·키즈카페)** -- Phase 3.

## 7. [NEEDS CLARIFICATION]

> 아직 결정되지 않은 사항. 개발 전에 정해야 합니다.

- [ ] **정책 데이터 소스 확정**: 공공데이터포털에 "정부 보조금24" API가 있는지, 자치구 정책은 어떻게 수집할지(크롤링/수동/계약). MVP는 수동 + 부분 API + 점진 확대 권장.
- [ ] **개인정보 처리방침 & 약관**: 소득·주택 정보 수집을 위해 별도 동의 흐름과 법적 검토 필요.
- [ ] **알림 시점 룰**: "D-7", "시기 도래" 등 정확한 트리거 규칙 명세화 (예: 임신 28주 = 산후조리비 지원 안내 시작).
- [ ] **시범 지역**: 전국 17개 광역을 한 번에 갈지, 서울특별시 우선 → 단계적 확대할지.
- [ ] **관리자 권한 모델**: 단일 관리자 vs 멀티 관리자 + 권한 분리.
- [ ] **소득분위 산정**: 사용자 입력값 신뢰 vs 별도 검증 절차(국세청 연동은 무리, 자기보고 + 안내 권장).
